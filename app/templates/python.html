Tutorial


    Basically, a Dockerfile is composed of a line-by-line instruction sequence. A line of instructions is a layer of data for an Image. An Image relies on such layers of data to accumulate, and finally compiles the desired image file. , like building a house. Explanation of the doctorfile parameter:

FROM: base image
Copy: various metadata of the source file will be preserved. Such as read, write, execute permissions, file change time, etc.
ENV: setting the environment of app, port and path WORKDIR: create a working directory CMD: There can only be one CMD instruction in a Dockerfile, and CMD is the instruction executed when the container is running.

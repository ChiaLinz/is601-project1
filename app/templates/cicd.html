Intro

  What is "continuous integration"?
  To ensure the quality of the program, there are some checks to be done before releasing the program. Hence, the existing program would not crush or have too many bug to fix when a new engineer join the project.
  The most basic thing is to confirm whether the code has syntax errors. Sometimes due to the urgency of time or careless negligence, the code with syntax errors may be released to the official environment, except for general syntax checks, the ideal program development process. In addition to implementing the required functions, engineers also need to write test code to test whether the program operates correctly. Before releasing the program, the test code should be executed to check whether the function code works as expected.
  Therefore, before we release the program, any inspection steps that need to be done, these steps are checked through the tool before going live, which is called "continuous integration".
  In short, CI/CD means that after the engineer has completed the development of the program, the developed code is subjected to a series of pre-launch inspections, and the code that passes the inspection can be directly released online. This process is the so-called CI/CD.

Why do we need "Continuous integration"?

  If the team today is small and the services provided are few, then after each development, manual publishing may not be a burden, but when the organization becomes larger, so does the service. More, in addition to regular updates, there may be online problems that need to be quickly launched after being dealt with. Under such a time-space background, manual publishing will be a bottleneck.
 In order to solve the trouble caused by manual release, the concept of "Continuous integration" was born accordingly.

Tutorial


First step:
  Go to Professor Keith Williams' github
clone the project, which could help to build the CI/CD. Second step:
  Go to Heroku sign up and create your own app, getting the API key.
Third step:
  Go to your github project, click settings, Security->Actions, and create three new repository secret. Including, "HEROKU_API_KEY" with the API key generate from the Heroku, "DOCKER_USERNAME" with your docker username/mail, and DOCKER_PASSWORD with your docker password.
Forth step:
  Go to your Heroku app dashboard -> settings and link the app to your github project.
Fifth step:
  Follow the instructions in the github to add the info to the files.
